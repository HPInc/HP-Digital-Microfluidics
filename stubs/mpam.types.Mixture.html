mpam.types.Mixture

class mpam.types.Mixture(name: str, mixture: Tuple[tuple['Reagent', fractions.Fraction], ...], *, composition: Optional[Mapping[Chemical, Union[MolarConcentration, Density, VolumeConcentration, UnknownConcentration]]] = None, min_storage_temp: Optional[TemperaturePoint] = None, max_storage_temp: Optional[TemperaturePoint] = None)

Bases: Reagent

A Reagent that is a mixture of two or more other Reagents

Mixture objects are not typically created directly. If one is needed as a Reagent, you can use find_or_compute() which computes (or finds a cached version of) a mixture between to Reagents. More commonly, Mixture objects will be created as a consequence of mixing together two Liquid objects using Liquid.mix_with(), Liquid.mix_in(), or Liquid.mix_together().

When created using one of the aforementioned methods, if the name of the resulting Mixture is not specified a name describing the relative proportions of the constituents will be constructed. For example, after

r3 = Mixture.find_or_compute(r1, r2, ratio = 0.4)

the name of r3 will be "2 r1 + 5 r2"

classmethod find_or_compute(r1: Reagent, r2: Reagent, *, ratio: float = 1, name: Optional[str] = None) → Reagent

A mixture of two Reagents in a given ratio.

The composition of the resulting Reagent will be computed based on the same ratio of the compositions of r1 and r2.

If name is not provided, a name describing the relative proportions of the constituents will be constructed. For example, after

r3 = Mixture.find_or_compute(r1, r2, ratio = 0.4)

the name of r3 will be "2 r1 + 5 r2"

Important

find_or_compute() is more efficient than new_mixture() and is the method that should be used. (find_or_compute() uses new_mixture() internally if necessary.) This is because in addition to caching results based on the constructed MixtureSpec, find_or_compute() caches them based on the arguments, so if the same Reagents are mixed repeatedly in the same proportions, the results will be found without having to compute the MixtureSpec.

Note

A ratio of n means n parts r1 to one part r2. This means that a ratio of 1 means equal parts, and a ratio of, e.g., 0.4 means two parts r1 to five parts r2.

Note

The results of calling find_or_compute() are cached, and if the same resulting MixtureSpec is discovered, the prior result will be returned. When mixtures are further mixed, this can happen even if a different path is taken

Parameters

  • r1 – the first Reagent

  • r2 – the second Reagent

Keyword Arguments

  • name – an optional name of the result.

property mixture: Tuple[tuple['Reagent', fractions.Fraction], ...]

A description of the Reagent as a mixture of other Reagents. The description is a tuple of tuples, each of which contains a Reagent and a Fraction specifying how much of the overall mixture that Reagent constitutes.

The elements of this tuple will be sorted by Reagent name, rendering the whole thing useful as a mapping key.

This Reagent is_pure() if and only if the the resulting tuple has one element, and the first element of that element will be this Reagent.

The second elements of each element of this tuple (Fractions) will all add up to 1.
