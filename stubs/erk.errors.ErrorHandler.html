erk.errors.ErrorHandler

class erk.errors.ErrorHandler

Bases: abc.ABC

An handler that can be used to handle (or check and, if necessary, handle) error conditions.

Example

The following definition of factorial checks to make sure that the argument is not negative, allowing the caller to specify the action if it is. By default, a warning will be printed, but this could be changed to having the code raise an exception or take other action.

def lookup(index: int, on_out_of_range: ErrorHandler = PRINT) -> str:
    on_out_of_range.expect_true(index < self.size,
                                lambda: f"No element {index}. Only {self.size} elements")
    return self.content[index]

All subclasses of ErrorHandler must define __call__(), which is passed in a string describing the error. ErrorHandler itself defines expect_true() and expect_false(), which take a boolean and a Callable that returns a string. If the boolean argument is False (for expect_true()) or True (for expect_false()), the Callable argument is evaluated and passed to __call__().

Defined subclasses of ErrorHandler include

DoNothing

Do nothing. The condition is still evaluated, but it isn’t checked and the message is never computed. This is typically used via its instance IGNORE:

s = obj.lookup(n, on_out_of_range = IGNORE)
PRINT_TO (where: TextIO )

Print the message to where. Predefined objects exist for printing to sys.stdout (PRINT) and sys.stderr (PRINT_TO_STDERR)

RAISE (factory)

Raise the exception returned by calling factory. Typically, this will be the name of the exception class, e.g.,

s = obj.lookup(n, on_out_of_range = RAISE(OutOfRangeError))
FIX_BY (function)

Call function (ignoring the message), which should, as a side effect, fix the problem. For example,

s = obj.lookup(n, on_out_of_range = FIX_BY(lambda: obj.grow_to(1.5*n)))
