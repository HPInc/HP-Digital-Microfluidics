mpam.device.Heater

class mpam.device.Heater(num: int, board: mpam.device.Board, *, pads: Sequence[mpam.device.Pad], polling_interval: quantities.dimensions.Time, initial_temperature: Optional[quantities.temperature.TemperaturePoint] = None)

Bases: mpam.device.BinaryComponent[Heater]

A BoardComponent that can be used to control the temperature of its associated pads.

The Heater is controlled by setting its target temperature. If its current_temperature is below its target, it attempts to become hotter until it reaches the target, at which point it attempts to maintain that temperature. If its current_temperature is above its target, it turns off and cools off until it reaches its target, at which point it attempts to maintain that temperature. If its target is None, it turns off and cools down to the ambient temperature.

As a BinaryComponent, its current_state is OFF whenever its target is None and ON otherwise. Setting its current_state to OFF is equivalent to setting its target to None. Setting its current_state to a TemperaturePoint is equivalent to setting its target to its last non-None value. (If there is no such value, its current_state will be reset to OFF.)

Note

The value of a Heater’s current_state is based on its target, not the state of its underlying hardware, so it remains ON when it is cooling to a target temperature or when it turns off briefly while maintaining a temperature. It is also OFF when its target is None, even if it has not yet cooled to the ambient temperature.

A Heater’s public interface is primarily three MonitoredPropertys, in addition to current_state. Its target is the temperature it is to get to and maintain, or None to indicate that it should turn off. Its read-only current_temperature is the last read (or modeled) temperature. And its read-only mode is an indication of whether it is MAINTAINING its target, HEATING to its target, COOLING to its target (or to the ambient temperature if its target is None), or OFF (and at the ambient temperature). These each have associated CallbackChangeLists: on_state_change, on_target_change, on_temperature_change, and on_mode_change.

Note

current_temperature can be None. This indicates that the Heater implementation has no way to read the temperature (or has not yet done so). If this is the case, the implementation will have to update mode itself, as the automatic updating is by callbacks hung on on_target_change and on_temperature_change.

The actual control of the hardware is expected to be a consequence of the the implementation subclass hanging a callback on on_target_change (and perhaps on_mode_change or on_temperature_change). The current_temperature is set to the value posted to the result of poll(), which is called every polling_interval. By default, it simply returns a future whose value is None, but this method is expected to be overridden by implementation subclasses. The polling can be stopped by stop_polling() and restarted by start_polling(). It is started as soon as the Heater joins the System.

Temperature changes on a Heater take place within a UserOperation, so the System will not be considered to be idle (and will not exit a with system block) when the Heater’s mode is HEATING or COOLING.

To set the target using an Operation, the Heater.SetTemperature operation is provided. This Operation completes when the desired temperature has been reached.

class SetTemperature(target: Optional[quantities.temperature.TemperaturePoint])

Bases: mpam.types.Operation[Heater, Heater]

A Operation that modifies the target of a Heater. The Operation is considered complete (and its future receives the Heater as its value) when the Heater’s current_temperature reaches this target temperature. (More precisely, when its mode becomes MAINTAINING or OFF.

Note

If another SetTemperature operation is scheduled before this one completes, this operation is cancelled, and its future will never receive a value.

property current_temperature: Optional[quantities.temperature.TemperaturePoint]

The current TemperaturePoint of the Heater. This is set to the value (posted to the future) returned by poll(). If this value is None, it means that no temperature has been established yet or the device cannot read its own temperature.

Note

If, for some reason, an implementation needs to set this at some other time, the actual MonitoredProperty is _temperature

property mode: mpam.device.HeatingMode

The Heater’s current HeatingMode. If this is HEATING or COOLING, the Heater is moving toward its target (or the ambient temperature if the target is None). If it is HeatingMode.MAINTAINING, the Heater has reached its target and is attempting to maintain the temperature there. And if it is HeatingMode.OFF, the target is None and the Heater has cooled down to ambient temperature.
