mpam.types.Operation

class mpam.types.Operation

Bases: Generic[mpam.types.T, mpam.types.V], abc.ABC

An operation that can be scheduled for an object of type T and returns a delayed value of type V

The basic notion is that if op is an Operation[T, V] and t is a T, in

dv: Delayed[V] = op.schedule_for(t)

the call will return immediately, and dv will obtain a value of type V at some point in the future, when the operation has completed.

If T is a subclass of OpScheduler, the same effect can be obtained by calling its schedule() method:

dv: Delayed[V] = t.schedule(op)

Note

Operation[T, V] is an abstract base class. The actual implementation class must implement _schedule_for().

Parameters

  • T – the type of the object used to schedule the Operation

  • V – the type of the value produced by the operation

schedule_for(obj: Union[mpam.types.T, mpam.types.Delayed[mpam.types.T]], *, after: Optional[Union[mpam.types.Ticks, quantities.dimensions.Time]] = None, post_result: bool = True) → mpam.types.Delayed[mpam.types.V]

Schedule this operation for the given object.

If obj is a Delayed object, the actual scheduling will take place after a value has been posted to it, and that value will be used. Note that any delay specified by after will be applied after this value is obtained.

Once an object has been identified, the actual scheduling will be delegated through _schedule_for().

Chain this Operation and another together to create a single new Operation

The value produced by this Operation will be used to schedule the second one (unless the second is a StaticOperation, in which case the second will be scheduled after this one is done).

The actual result will be a CombinedOperation[T, V, V2].

Note

If op is a Callable, it will be evaluated when the Operation that is the result of then() is scheduled, not when this Operation produces a value.
