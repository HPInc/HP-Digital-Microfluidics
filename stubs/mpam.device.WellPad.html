mpam.device.WellPad

class mpam.device.WellPad(board: Board, state: State[OnOff], *, live: bool = True, neighbors: Sequence[int])

Bases: BinaryComponent[WellPad], DropLoc

A pad inside a :class:Well`.

  • As a BinaryComponent, it has a board and a current_state, it can act as both a CommunicationScheduler and a OpScheduler[Pad], and it supports on_state_change().

    To initialize this asspect of the Pad, when it is created, it must be associated with a State[OnOff] object that knows how to effect any changes on a physical device (via its realize_state() method). If nothing needs to be done, a DummyState may be used.

  • As a DropLoc, it had an optional drop and blob, and it supports on_drop_change(). It is empty whenever drop is None.

A WellPad that is its well’s gate will be an instance of the subclass WellGate. The attribute is_gate can be used to distinguish gates from interior WellPads.

Each Well had a list of shared_pads that enumerates the the interior pads inside it. A WellPad’s index specifies its position in this list. If it is a WellGate, index will be -1. This attribute is set by means of set_location(), called within Well.__init__(). (The WellPad`s are created before the :class:`Well and passed into its __init__() method, which calls set_location() to establish well and index.)

Whenever the current_state changes, board’s journal_state_change() is called with the (necessarily different) old and new values to prep for drop motion inference.

compute_neighbors_for_blob() → Sequence[DropLoc]

The list of neighboring DropLocs that could be part of the same Blob. This is called the first time neighbors_for_blob is referenced, and the value is cached.

This will be the WellPads corresponding to the neighbors indices provided to __init__(). If is_gate is True (i.e., this is a WellGate), it will also include well’s exit_pad.
