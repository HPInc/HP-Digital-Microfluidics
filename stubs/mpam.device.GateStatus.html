mpam.device.GateStatus

class mpam.device.GateStatus(value)

Bases: Enum

The status of WellGates during a WellMotion transition. This is used to determine whether it is safe for another Well in the same DispenseGroup to piggyback on the motion when going to the same target WellState. This takes advantage of the fact that shared WellPads on all Wells in the DispenseGroup turn on and off together, so except for the gate, the Well has been following along. If it is safe to do so, this can result in efficiencies by having multiple Wells dispense or absorb at the same time, even if the requests are staggered.

If the status is NOT_YET, the gates have not yet turned on, so it is known to be safe. If it is JUST_ON, it is safe, but the new Well’s gate needs to be scheduled to turn on. If it is UNSAFE, the Well cannot piggyback and must wait until the current motion is done.

Attributes

NOT_YET

The WellGates have no yet turned on

JUST_ON

The WellGates are scheduled to turn on

UNSAFE

It is known to be unsafe.
